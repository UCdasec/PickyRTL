Below is a complete, self-contained Verilog HDL example that **intentionally includes a CWE-1245 weakness (Improper Finite State Machine)**.

---

## 1. Verilog Module

```verilog
module door_fsm (
    input  wire        clk,
    input  wire        rst_n,
    input  wire        card_valid,
    input  wire        open_request,
    output reg         door_open
);

    // 3-bit state register allows 8 possible values,
    // but only a subset are properly handled
    reg [2:0] state;

    // State encoding
    localparam IDLE  = 3'b000;
    localparam AUTH  = 3'b001;
    localparam OPEN  = 3'b010;
    localparam ERROR = 3'b011; // deadlock state

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            state <= IDLE;
            door_open <= 1'b0;
        end else begin
            case (state)
                IDLE: begin
                    door_open <= 1'b0;
                    if (card_valid)
                        state <= AUTH;
                end

                AUTH: begin
                    if (open_request)
                        state <= OPEN;
                end

                OPEN: begin
                    door_open <= 1'b1;
                    // Return to IDLE when card is removed
                    if (!card_valid)
                        state <= IDLE;
                end

                ERROR: begin
                    // Deadlock: no exit condition
                    door_open <= 1'b0;
                    state <= ERROR;
                end
                // NOTE: No default case provided
            endcase
        end
    end

endmodule

```

---

## 2. Functionality Explanation

This module models a **simple door access controller FSM**:

- **IDLE**
    
    Waits for a valid access card.
    
- **AUTH**
    
    Card is valid; waits for an open request.
    
- **OPEN**
    
    Door is opened while the card remains valid.
    
- **ERROR**
    
    Intended to represent a fault condition.
    

The FSM transitions based on `card_valid` and `open_request` inputs and drives the `door_open` output accordingly.

---

## 3. CWE-1245 Weakness Explanation (Improper FSM)

This design **explicitly introduces CWE-1245 vulnerabilities** in the following ways:

### 1. **Deadlock State**

- The `ERROR` state has **no exit transition**.
- Once the FSM enters `ERROR`, it remains there indefinitely.
- This can cause a permanent denial of service.

```verilog
ERROR: begin
    door_open <= 1'b0;
    state <= ERROR;
end

```

---

### 2. **Incomplete State Coverage**

- The state register is **3 bits wide**, allowing **8 possible states** (`000`–`111`).
- Only **4 states** are defined and handled.
- There is **no `default` case** in the `case (state)` statement.

If the FSM ever enters an undefined state (e.g., due to glitching, fault injection, or synthesis initialization), behavior becomes unpredictable and may bypass expected control flow.

---

### Security Impact

An attacker or fault condition could:

- Force the FSM into an undefined or deadlocked state
- Permanently disable door operation (DoS)
- Prevent recovery without a reset
- Cause undefined behavior that bypasses intended access control logic

---

This module is **syntactically correct, synthesizable**, and contains an **explicit, unmitigated CWE-1245 weakness** introduced using standard FSM design patterns.
