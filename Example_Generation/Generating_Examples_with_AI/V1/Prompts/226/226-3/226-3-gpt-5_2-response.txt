Below is a deliberately vulnerable, **syntactically correct and synthesizable Verilog HDL module** that demonstrates **CWE-226: Sensitive Information in Resource Not Removed Before Reuse**.

---

## 1. Complete Verilog Module

```verilog
module leaky_fifo #(
    parameter DEPTH = 8,
    parameter WIDTH = 8
)(
    input  wire              clk,
    input  wire              rst,
    input  wire              wr_en,
    input  wire              rd_en,
    input  wire [WIDTH-1:0]  din,
    output reg  [WIDTH-1:0]  dout
);

    // Internal memory reused across resets
    reg [WIDTH-1:0] mem [0:DEPTH-1];

    // Read/write pointers
    reg [$clog2(DEPTH)-1:0] rd_ptr;
    reg [$clog2(DEPTH)-1:0] wr_ptr;

    always @(posedge clk) begin
        if (rst) begin
            // Reset only control state
            rd_ptr <= 0;
            wr_ptr <= 0;
            dout   <= 0;
            // NOTE: mem[] is intentionally NOT cleared
        end else begin
            if (wr_en) begin
                mem[wr_ptr] <= din;
                wr_ptr <= wr_ptr + 1;
            end

            if (rd_en) begin
                dout <= mem[rd_ptr];
                rd_ptr <= rd_ptr + 1;
            end
        end
    end

endmodule

```

---

## 2. Module Functionality Explanation

This module implements a simple synchronous FIFO-style buffer:

- `mem` is an internal memory array used to store data
- `wr_ptr` and `rd_ptr` track write and read positions
- When `wr_en` is asserted, input data (`din`) is written into memory
- When `rd_en` is asserted, data is read from memory and presented on `dout`
- On reset (`rst`), the FIFO control logic is reset so that read and write start again from index `0`

From a functional standpoint, this behaves like a small reusable data buffer that can be reset and reused across multiple operational phases.

---

## 3. CWE-226 Weakness Explanation

### Where the Weakness Occurs

The CWE-226 weakness is introduced in the **reset logic**:

```verilog
if (rst) begin
    rd_ptr <= 0;
    wr_ptr <= 0;
    dout   <= 0;
    // mem[] is NOT cleared
end

```

### How the Weakness Manifests

- The FIFO’s internal memory (`mem`) may contain **sensitive data** written during a previous operational phase (e.g., credentials, keys, private data).
- When the module is reset, **only the control pointers are reset**, not the memory itself.
- After reset, subsequent reads (`rd_en`) will access memory locations that still contain **residual data from before the reset**.
- This results in **previously stored sensitive information being exposed** to a new user, session, or privilege context.

### Why This Is CWE-226

CWE-226 is defined as *“Sensitive Information in Resource Not Removed Before Reuse”*.

In this design:

- The memory array is a **reused resource**
- Sensitive values persist across reset boundaries
- The design explicitly allows retrieval of stale data
- No clearing or sanitization logic is present

This creates a **confidentiality violation** where residual data can be recovered after the resource is logically released and reused.
