Below is a **self-contained, synthesizable Verilog HDL module** that **intentionally includes a CWE-226 weakness**. The vulnerability is present in active logic and is **not mitigated**.

---

## 1. Complete Verilog Module

```verilog
// Intentionally vulnerable example demonstrating CWE-226
// Sensitive Information in Resource Not Removed Before Reuse

module session_key_processor (
    input  wire        clk,
    input  wire        reset_n,      // Active-low reset
    input  wire        start_session, // Begins a new session
    input  wire        end_session,   // Ends the current session
    input  wire        load_key,      // Loads a sensitive key
    input  wire [31:0] key_in,        // Sensitive key input
    input  wire [31:0] data_in,
    output reg  [31:0] data_out,
    output reg         busy
);

    // FSM states
    localparam IDLE   = 2'b00;
    localparam ACTIVE = 2'b01;

    reg [1:0]  state;
    reg [31:0] secret_key;  // Sensitive resource (cryptographic key)

    // Sequential logic
    always @(posedge clk or negedge reset_n) begin
        if (!reset_n) begin
            // Reset state but DO NOT clear secret_key (intentional weakness)
            state <= IDLE;
            busy  <= 1'b0;
        end else begin
            case (state)
                IDLE: begin
                    busy <= 1'b0;
                    if (start_session) begin
                        state <= ACTIVE;
                        busy  <= 1'b1;
                    end
                end

                ACTIVE: begin
                    if (load_key) begin
                        // Load sensitive information
                        secret_key <= key_in;
                    end

                    if (end_session) begin
                        // End session but DO NOT clear secret_key
                        state <= IDLE;
                        busy  <= 1'b0;
                    end
                end
            endcase
        end
    end

    // Simple data processing using the secret key
    always @(*) begin
        if (state == ACTIVE) begin
            data_out = data_in ^ secret_key;
        end else begin
            data_out = 32'b0;
        end
    end

endmodule

```

---

## 2. Module Functionality Explanation

This module models a **simple session-based data processor**:

- A session begins when `start_session` is asserted.
- During an active session:
    - A **sensitive 32-bit key** (`secret_key`) may be loaded using `load_key`.
    - Incoming data is XORed with the secret key and presented on `data_out`.
- When `end_session` is asserted, the module returns to the idle state.
- The `busy` signal indicates whether a session is active.

This structure resembles a simplified cryptographic or secure-processing block often found in hardware designs.

---

## 3. CWE-226 Weakness Explanation

### Where the Weakness Is Introduced

The CWE-226 weakness is explicitly introduced in **two places**:

1. **During Reset**
    
    ```verilog
    if (!reset_n) begin
        state <= IDLE;
        busy  <= 1'b0;
        // secret_key is NOT cleared here
    end
    
    ```
    
    The `secret_key` register retains its previous value even after a reset.
    
2. **During Session Termination**
    
    ```verilog
    if (end_session) begin
        state <= IDLE;
        busy  <= 1'b0;
        // secret_key is NOT cleared here
    end
    
    ```
    
    When a session ends, the sensitive key remains stored in the register.
    

---

### How This Maps to CWE-226

- `secret_key` is a **resource containing sensitive information**.
- The resource is **reused across sessions**.
- The sensitive data is **not cleared before reuse or release**.
- A subsequent session (or different security context) can potentially:
    - Access residual key material
    - Infer previous session secrets

---

### Security Impact

This violates confidentiality expectations and can lead to:

- Leakage of cryptographic keys
- Recovery of sensitive data from prior sessions
- Cross-session or cross-privilege information exposure

This is a **textbook example of CWE-226: Sensitive Information in Resource Not Removed Before Reuse**, intentionally left unmitigated as requested.
