CWE-1431: Driving Intermediate Cryptographic State/Results to Hardware Module Outputs

Hardware-based cryptographic modules often maintain intermediate states or results while performing cryptographic operations. These intermediate values are highly sensitive and should not be leaked to unauthorized parties.

Vulnerability Pattern:
This vulnerability happens when intermediate cryptographic results are driven to module outputs before the cryptographic operation is complete.

Security Impact:
Cryptographic algorithms need to be implemented correctly to maintain security guarantees. If intermediate encryption data is leaked and observed by an untrusted party, it might be possible for that untrusted party to identify secret information (such as keys) used within the cryptographic algorithm.
