## 1. Complete Verilog Module

```verilog
module auth_controller_fsm (
    input  wire        clk,
    input  wire        rst_n,
    input  wire        start_auth,
    input  wire        password_valid,
    input  wire [1:0]  privilege_req,
    input  wire        timeout_tick,

    output reg         auth_granted,
    output reg  [1:0]  access_level,
    output reg         busy,
    output reg         error_flag
);

    // ===============================
    // Authentication FSM (Main FSM)
    // ===============================
    typedef enum reg [2:0] {
        IDLE        = 3'b000,
        INIT        = 3'b001,
        COLLECT     = 3'b010,
        VERIFY      = 3'b011,
        GRANT       = 3'b100,
        DENY        = 3'b101,
        LOCKED      = 3'b110   // Deadlock state (intentional)
        // 3'b111 is not defined (incomplete state coverage)
    } auth_state_t;

    auth_state_t auth_state;

    // ===============================
    // Key Handling FSM (Secondary FSM)
    // ===============================
    typedef enum reg [1:0] {
        KEY_IDLE    = 2'b00,
        KEY_LOAD    = 2'b01,
        KEY_CHECK   = 2'b10
        // 2'b11 unreachable
    } key_state_t;

    key_state_t key_state;

    // ===============================
    // Timeout Counter
    // ===============================
    reg [3:0] timeout_cnt;

    // ===============================
    // Authentication FSM Logic
    // ===============================
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            auth_state   <= IDLE;
            auth_granted <= 1'b0;
            access_level <= 2'b00;
            busy         <= 1'b0;
            error_flag   <= 1'b0;
            timeout_cnt  <= 4'd0;
        end else begin
            case (auth_state)
                IDLE: begin
                    busy <= 1'b0;
                    if (start_auth) begin
                        auth_state <= INIT;
                        busy       <= 1'b1;
                    end
                end

                INIT: begin
                    timeout_cnt <= 4'd0;
                    auth_state  <= COLLECT;
                end

                COLLECT: begin
                    if (timeout_tick)
                        timeout_cnt <= timeout_cnt + 1'b1;

                    if (timeout_cnt == 4'd10)
                        auth_state <= LOCKED; // Forced deadlock
                    else
                        auth_state <= VERIFY;
                end

                VERIFY: begin
                    if (password_valid)
                        auth_state <= GRANT;
                    else
                        auth_state <= DENY;
                end

                GRANT: begin
                    auth_granted <= 1'b1;
                    access_level <= privilege_req;
                    auth_state   <= IDLE;
                end

                DENY: begin
                    error_flag <= 1'b1;
                    auth_state <= IDLE;
                end

                LOCKED: begin
                    // INTENTIONAL CWE-1245:
                    // No transition out of LOCKED
                    // System remains permanently stuck here
                    busy <= 1'b1;
                end

                // INTENTIONAL CWE-1245:
                // Missing default case leads to undefined behavior
                // if auth_state == 3'b111
            endcase
        end
    end

    // ===============================
    // Key Handling FSM Logic
    // ===============================
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            key_state <= KEY_IDLE;
        end else begin
            case (key_state)
                KEY_IDLE: begin
                    if (auth_state == INIT)
                        key_state <= KEY_LOAD;
                end

                KEY_LOAD: begin
                    key_state <= KEY_CHECK;
                end

                KEY_CHECK: begin
                    if (password_valid)
                        key_state <= KEY_IDLE;
                end
                // INTENTIONAL CWE-1245:
                // No default case and unreachable state (2'b11)
            endcase
        end
    end

endmodule
```

## 2. Module Functionality Explanation

**Application Domain:** Authentication / Authorization System

This module simulates a **security-critical authentication controller** used to validate a user and assign a privilege level.

### Key Features

* **Two interacting FSMs**

  * **Authentication FSM (7 states)** controlling login flow
  * **Key-handling FSM** simulating secret key preparation
* **Security-critical operation**

  * Authentication grant or denial
  * Privilege level assignment
* **Counter-based timeout**

  * Forces lockout after excessive delays
* **Multiple inputs and outputs**

  * Inputs: clock, reset, authentication start, password validity, privilege request, timeout tick
  * Outputs: authentication result, access level, busy flag, error flag

### High-Level Flow

1. System waits in `IDLE`
2. On `start_auth`, initializes authentication
3. Collects credentials with timeout monitoring
4. Verifies password
5. Grants or denies access
6. Can enter a locked state on timeout

## 3. CWE-1245 Weakness Explanation (Improper FSM)

This module **intentionally includes multiple CWE-1245 violations**:

### 🔴 1. Deadlock State (Critical)

* The `LOCKED` state **has no exit path**
* Once entered, the FSM **can never recover**, even if inputs change
* This enables **permanent denial-of-service**

```verilog
LOCKED: begin
    busy <= 1'b1;
end
```

---

### 🔴 2. Incomplete State Coverage

* `auth_state` is 3 bits wide → **8 possible values**
* Only **7 states are defined**
* State `3'b111` is **not handled**
* No `default` case exists

This allows:

* Undefined behavior
* FSM corruption due to glitches or fault injection

---

### 🔴 3. Unreachable State in Secondary FSM

* `key_state` has a valid encoding space of 4 values
* State `2'b11` is **never reachable**
* No default handling

---

## Security Impact

An attacker could exploit this improper FSM to:

* Permanently lock the system (`LOCKED` deadlock)
* Force undefined state transitions
* Cause denial of service
* Disrupt authentication logic
