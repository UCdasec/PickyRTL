CWE-226: Sensitive Information in Resource Not Removed Before Reuse

Hardware and software systems reuse resources such as memory, registers, and storage regions across different processes with different privilege levels or security levels. When a resource is released, any sensitive data it contains should be cleared before the resource is made available for reuse.

Vulnerability Pattern:

This weakness occurs when a resource containing sensitive information is not cleared before being released for reuse.  This manifests when a register retains sensitive values across module reset or critical state transitions. 

Security Impact:

Failure to remove sensitive information before resources are reused can lead to confidentiality violations. Untrusted actors may recover residual data from reused resources. This could expose cryptographic secrets or private user information.
