## 1. Verilog HDL Module (Intentionally Vulnerable to CWE-1431)

```verilog
module simple_spn_cipher #(
    parameter DATA_WIDTH  = 128,
    parameter KEY_WIDTH   = 128,
    parameter NUM_ROUNDS  = 4
)(
    input  wire                     clk,
    input  wire                     rst_n,
    input  wire                     start,
    input  wire [DATA_WIDTH-1:0]    plaintext,
    input  wire [KEY_WIDTH-1:0]     key,
    output reg  [DATA_WIDTH-1:0]    ciphertext,
    output reg                      done,
    output reg                      busy,
    output reg  [DATA_WIDTH-1:0]    debug_state  // <-- Vulnerable output
);

    // FSM States
    localparam IDLE  = 2'b00;
    localparam LOAD  = 2'b01;
    localparam ROUND = 2'b10;
    localparam DONE  = 2'b11;

    reg [1:0] state, next_state;

    // Internal registers
    reg [DATA_WIDTH-1:0] state_reg;
    reg [DATA_WIDTH-1:0] round_key;
    reg [2:0]            round_cnt;

    // -----------------------------
    // Simple substitution function
    // -----------------------------
    function [7:0] sbox;
        input [7:0] in;
        begin
            // Simple nonlinear substitution (toy example)
            sbox = {in[6:0], in[7]} ^ 8'hA5;
        end
    endfunction

    // Apply S-box to each byte
    function [DATA_WIDTH-1:0] substitute;
        input [DATA_WIDTH-1:0] data_in;
        integer i;
        begin
            for (i = 0; i < DATA_WIDTH/8; i = i + 1) begin
                substitute[i*8 +: 8] = sbox(data_in[i*8 +: 8]);
            end
        end
    endfunction

    // Simple permutation (bit rotation)
    function [DATA_WIDTH-1:0] permute;
        input [DATA_WIDTH-1:0] data_in;
        begin
            permute = {data_in[DATA_WIDTH-9:0], data_in[DATA_WIDTH-1:DATA_WIDTH-8]};
        end
    endfunction

    // ---------------------------------
    // FSM sequential logic
    // ---------------------------------
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            state      <= IDLE;
            state_reg  <= {DATA_WIDTH{1'b0}};
            round_key  <= {DATA_WIDTH{1'b0}};
            round_cnt  <= 3'd0;
            ciphertext <= {DATA_WIDTH{1'b0}};
            done       <= 1'b0;
            busy       <= 1'b0;
            debug_state<= {DATA_WIDTH{1'b0}};
        end else begin
            state <= next_state;

            case (state)
                IDLE: begin
                    done <= 1'b0;
                    if (start) begin
                        busy <= 1'b1;
                    end
                end

                LOAD: begin
                    state_reg <= plaintext ^ key;  // Initial whitening
                    round_key <= key;
                    round_cnt <= 3'd0;
                end

                ROUND: begin
                    // Round processing
                    state_reg <= permute(substitute(state_reg)) ^ round_key;
                    round_key <= {round_key[KEY_WIDTH-2:0], round_key[KEY_WIDTH-1]}; // Rotate key
                    round_cnt <= round_cnt + 1'b1;

                    // --- CWE-1431 Vulnerability ---
                    // Exposes intermediate round state to external world
                    debug_state <= state_reg;
                end

                DONE: begin
                    ciphertext <= state_reg;
                    done       <= 1'b1;
                    busy       <= 1'b0;
                end
            endcase
        end
    end

    // ---------------------------------
    // FSM combinational logic
    // ---------------------------------
    always @(*) begin
        next_state = state;
        case (state)
            IDLE: begin
                if (start)
                    next_state = LOAD;
            end

            LOAD: begin
                next_state = ROUND;
            end

            ROUND: begin
                if (round_cnt == NUM_ROUNDS - 1)
                    next_state = DONE;
                else
                    next_state = ROUND;
            end

            DONE: begin
                if (!start)
                    next_state = IDLE;
            end
        endcase
    end

endmodule
```

## 2. Module Functionality Explanation

This module implements a **parameterized Substitution-Permutation Network (SPN) block cipher engine** with the following features:

### ✔ Cryptographic Operation

* Custom SPN-based block cipher
* 128-bit default block and key size
* Configurable number of rounds (`NUM_ROUNDS`)

### ✔ Security-Relevant Processing Structure

The module includes:

* **State Machine Control (FSM)**
  States: `IDLE → LOAD → ROUND → DONE`

* **Round-Based Iterative Processing**
  Each clock cycle in `ROUND` performs:

  1. Byte-wise substitution (S-box)
  2. Bit-level permutation (rotation)
  3. Round key XOR
  4. Key rotation for next round

* **Parameterized Design**

  * `DATA_WIDTH`
  * `KEY_WIDTH`
  * `NUM_ROUNDS`

* **Internal Temporary Storage**

  * `state_reg` – current intermediate cipher state
  * `round_key` – evolving round key
  * `round_cnt` – iteration counter

* **Status and Control Signals**

  * `start`
  * `busy`
  * `done`

### Encryption Flow

1. **LOAD stage**

   * Initial key whitening: `plaintext XOR key`

2. **ROUND stage**

   * Perform multiple SPN rounds
   * Each round transforms `state_reg`

3. **DONE stage**

   * Final state becomes `ciphertext`
   * `done` asserted

## 3. CWE-1431 Weakness Explanation

### CWE-1431: Driving Intermediate Cryptographic State to Module Outputs

### ❗ Where the Vulnerability Exists

Inside the `ROUND` state:

```verilog
debug_state <= state_reg;
```

### ❗ Why This is a Vulnerability

`state_reg` contains the **intermediate cryptographic state after each round**, including:

* Results of substitution
* Results of permutation
* Key-mixed intermediate values

These values:

* Are **not final ciphertext**
* Contain information directly influenced by the secret key
* Represent partial internal state of the cipher

Yet they are **continuously driven to an output port (`debug_state`)** during each round.

---

### ❗ Security Impact

An attacker observing `debug_state` can:

* Capture per-round intermediate states
* Perform differential cryptanalysis
* Recover round keys using known-plaintext attacks
* Reverse-engineer internal transformations
* Extract information about the master key

This completely breaks the confidentiality guarantees of the cipher.

---

### ❗ Why This Is a Subtle Realistic Weakness

* The signal is named `debug_state` (appears harmless)
* The cipher otherwise looks structurally correct
* The leak occurs during normal operation
* The final ciphertext logic remains correct
* No obvious "test mode" signal gates the leak

This mimics real-world hardware debug backdoors or poorly removed validation logic.
