## 1. Complete Verilog Module (With Intentional CWE-1431 Weakness)

```verilog
module spn_cipher_with_leakage #
(
    parameter DATA_WIDTH  = 128,
    parameter KEY_WIDTH   = 128,
    parameter NUM_ROUNDS  = 8
)
(
    input                       clk,
    input                       rst_n,
    input                       start,
    input  [DATA_WIDTH-1:0]     plaintext_i,
    input  [KEY_WIDTH-1:0]      key_i,

    output reg [DATA_WIDTH-1:0] ciphertext_o,
    output reg                  valid_o,
    output reg                  busy_o,

    // *** Intentional CWE-1431 Weakness: intermediate state exposed ***
    output [DATA_WIDTH-1:0]     round_state_o
);

    // FSM States
    localparam S_IDLE    = 2'd0;
    localparam S_PROCESS = 2'd1;
    localparam S_DONE    = 2'd2;

    reg [1:0] state_q, state_d;

    // Round counter
    reg [$clog2(NUM_ROUNDS+1)-1:0] round_cnt_q, round_cnt_d;

    // Internal state registers
    reg [DATA_WIDTH-1:0] state_q_reg, state_d_reg;
    reg [KEY_WIDTH-1:0]  key_q_reg;

    // Next round key (simple rotation-based derivation)
    wire [KEY_WIDTH-1:0] round_key_w;
    assign round_key_w = {key_q_reg[KEY_WIDTH-9:0], key_q_reg[KEY_WIDTH-1:KEY_WIDTH-8]} 
                         ^ round_cnt_q;

    // --- Substitution layer (simple nibble inversion as example S-box) ---
    function [DATA_WIDTH-1:0] sub_layer;
        input [DATA_WIDTH-1:0] data_in;
        integer i;
        begin
            for (i = 0; i < DATA_WIDTH; i = i + 4) begin
                sub_layer[i +: 4] = ~data_in[i +: 4];
            end
        end
    endfunction

    // --- Permutation layer (simple rotate-left by 3 bits) ---
    function [DATA_WIDTH-1:0] perm_layer;
        input [DATA_WIDTH-1:0] data_in;
        begin
            perm_layer = {data_in[DATA_WIDTH-4:0], data_in[DATA_WIDTH-1:DATA_WIDTH-3]};
        end
    endfunction

    // Combinational next-state logic
    always @(*) begin
        state_d        = state_q;
        round_cnt_d    = round_cnt_q;
        state_d_reg    = state_q_reg;

        case (state_q)
            S_IDLE: begin
                if (start) begin
                    state_d     = S_PROCESS;
                    round_cnt_d = 0;
                    state_d_reg = plaintext_i ^ key_i; // Initial whitening
                end
            end

            S_PROCESS: begin
                if (round_cnt_q < NUM_ROUNDS) begin
                    state_d_reg = perm_layer(sub_layer(state_q_reg)) ^ round_key_w;
                    round_cnt_d = round_cnt_q + 1;
                end
                else begin
                    state_d = S_DONE;
                end
            end

            S_DONE: begin
                state_d = S_IDLE;
            end

            default: begin
                state_d = S_IDLE;
            end
        endcase
    end

    // Sequential logic
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            state_q       <= S_IDLE;
            round_cnt_q   <= 0;
            state_q_reg   <= 0;
            key_q_reg     <= 0;
            ciphertext_o  <= 0;
            valid_o       <= 0;
            busy_o        <= 0;
        end
        else begin
            state_q     <= state_d;
            round_cnt_q <= round_cnt_d;
            state_q_reg <= state_d_reg;

            case (state_q)
                S_IDLE: begin
                    valid_o <= 0;
                    busy_o  <= 0;
                    if (start) begin
                        key_q_reg <= key_i;
                        busy_o    <= 1;
                    end
                end

                S_PROCESS: begin
                    busy_o  <= 1;
                    valid_o <= 0;
                end

                S_DONE: begin
                    ciphertext_o <= state_q_reg;
                    valid_o      <= 1;
                    busy_o       <= 0;
                end
            endcase
        end
    end

    // ================================================================
    // CWE-1431 WEAKNESS:
    // Directly exposing intermediate round state to module output
    // ================================================================
    assign round_state_o = state_q_reg;

endmodule
```

## 2. Module Functionality Explanation

This module implements a **parameterized Substitution-Permutation Network (SPN) block cipher engine** with the following features:

### ✔ Parameterized Design

* `DATA_WIDTH` (default 128 bits)
* `KEY_WIDTH` (default 128 bits)
* `NUM_ROUNDS` (default 8 rounds)

### ✔ State Machine Control (FSM)

Three states:

* `S_IDLE` – Waits for `start`
* `S_PROCESS` – Performs iterative encryption rounds
* `S_DONE` – Outputs final ciphertext and asserts `valid_o`

### ✔ Round-Based Iterative Processing

Each round performs:

1. **Substitution layer** – 4-bit nibble inversion (example S-box)
2. **Permutation layer** – 3-bit left rotation
3. **Round key mixing** – XOR with a rotated key

### ✔ Input/Output Buffering

* `plaintext_i` captured into internal register on `start`
* `key_i` latched into `key_q_reg`
* `ciphertext_o` updated only when encryption completes
* `valid_o` indicates completion
* `busy_o` indicates active processing

### ✔ Internal Temporary Storage

* `state_q_reg` holds intermediate round state
* `round_cnt_q` tracks iteration count
* `key_q_reg` stores working key material

## 3. CWE-1431 Weakness Explanation

### Vulnerability: Driving Intermediate Cryptographic State to Output

The weakness is introduced here:

```verilog
assign round_state_o = state_q_reg;
```

### What This Does

`state_q_reg` holds the **internal encryption state after each round**.

During `S_PROCESS`, this register contains:

* Partially encrypted data
* Round-by-round transformed internal state
* Values that depend on both plaintext and secret key

However, this internal state is continuously driven to the external output:

```
output [DATA_WIDTH-1:0] round_state_o;
```

### Why This Is CWE-1431

CWE-1431 occurs when:

> Intermediate cryptographic results are driven to module outputs before the cryptographic operation is complete.

Here:

* The encryption core is mid-computation
* Sensitive round state is externally observable
* An attacker can monitor intermediate transformations
* Differential or side-channel style analysis becomes easier
* Secret key information may be derivable

### Security Impact

Exposing round state enables:

* Round-by-round cryptanalysis
* Key recovery via known-plaintext observation
* Reduction of effective cipher strength

In a secure design, `state_q_reg` would remain strictly internal and never be exposed.
