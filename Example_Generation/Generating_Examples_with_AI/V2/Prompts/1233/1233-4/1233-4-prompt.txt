CWE-1233: Security-Sensitive Hardware Controls with Missing Lock Bit Protection

Security-sensitive hardware registers often control critical system behavior, such as device configuration, debug access, privilege escalation paths, or storage of trusted keys. To protect these registers from being modified after initial configuration, hardware designs implement a lock bit. The lock bit is intended to prevent any unauthorized writes to the protected register

Vulnerability pattern:

This weakness occurs when hardware logic allows writes to security-critical registers without first checking the lock bit. Correct behavior requires that if the lock bit is high, any write attempt is rejected/ignored, and if the lock bit is low, any write is permitted. This weakness exists when the check is missing, which allows unauthorized writes.

                endcase

                4:
