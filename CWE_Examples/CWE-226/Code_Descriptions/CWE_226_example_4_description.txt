This code example is from the HMAC wrapper of the HACK@DAC'21 OpenPiton SoC repository. 
It was inserted as bug #97 for CWE-1239, but since CWE-1239 is a child of CWE-226 it 
can serve as a good example of CWE-226. In the code startHash, newMessage, and data 
registers are cleared on reset. However other registers such as key0 registers are 
not cleared on reset. The weakness is introduced when key data is left on the registers 
and could be attained by an untrusted agent. This key could be used to gain access to 
security sensitive data.
